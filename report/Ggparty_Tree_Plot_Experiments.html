1 Purpose

This notebook tests ggplot-based tree rendering from the saved DRC results object. The existing ci_report_tree_plot() uses the package/base plotting method. The experimental ci_tree_gg() helper uses ggparty, which means the tree can be styled with ordinary ggplot2 layers and can optionally use ggrepel::geom_label_repel() when ggrepel is installed.

  terminal_text_size = 2.5,
  edge_text_size = 2.5,
  split_label_width = 28,

  terminal_label_width = 18,
  terminal_layout = "level",
  node_box_style = "ci_report_tree",
  use_repel = repel_mode
)
terminal_depths <- function(fit, label) {
  p <- ci_tree_gg(
    fit = fit,
    data = selected_forest_surrogate$data,
    outcome_name = selected_forest_surrogate$prediction_name,
    outcome_label = "U5",
    ci_type = selected_tree_type,
    var_labels = congo_var_labels,
    terminal_layout = "level"
  )
  depth_dt <- as.data.table(p$data)[kids == 0L]
  depth_dt[, plot := label]
  depth_dt[, .N, by = .(plot, true_level)][
    order(plot, true_level)
  ]
}

terminal_depths(
  selected_forest_surrogate$fit,
  "Selected forest surrogate"
)

5 Notes

The ggparty layout uses the tree’s node structure but does not promise pixel identity with the base partykit plot. Its advantage is that the returned object is a normal ggplot, so labels, margins, text sizes, fills, and optional repelling can be tuned in the usual ggplot ecosystem.
